# Adjacent Skills

Use these suggestions after the main answer, not before the user gets value.

## Routing table

- User asks for strategic diagnosis, annual BP, executive strategy memo:
  Recommend `huawei-strategy-principles` first if the logic is weak, then `strategy-blm-dste-execution`.
- User asks for market segmentation, target market selection, opportunity ranking:
  Recommend `huawei-market-principles` first if the logic is weak, then `market-management-execution`.
- User asks for roadmap critique, release logic, product line tradeoff:
  Recommend `huawei-product-principles` first if planning quality is weak, then `product-planning-execution`.
- User asks for roadmap, product line planning, version goals, release package:
  Recommend `product-planning-execution`.
- User asks for development execution, stage gates, milestone plan, review checklist:
  Recommend `product-development-ipd-execution`.
- User asks for lead flow, opportunity governance, contract-to-delivery-to-cash:
  Recommend `sales-ltc-execution`.
- User asks for service desk, issue closure, escalation mechanism, VOC feedback loop:
  Recommend `huawei-service-principles` first if the service design is weak, then `service-itr-execution`.
- User asks for method rollout, PMO, governance cadence, pilot and scale:
  Recommend `transformation-pmo-execution`.

## Multi-stage sequences

- `Strategy -> Product planning -> IPD execution`
  Use when the user starts with business strategy and wants to land it in product delivery.
- `Market -> Product planning -> IPD execution`
  Use when the user starts from segment and opportunity decisions and wants to land them in roadmap and delivery.
- `Sales -> Service -> Strategy`
  Use when the user wants revenue flow, customer issue closure, and strategic feedback integrated.
- `Strategy -> Transformation`
  Use when the user wants governance, rollout, or organizational adoption.
- `Product planning -> IPD execution`
  Use when the strategy already exists and the remaining work is delivery-oriented.
