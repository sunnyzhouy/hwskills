# Workflow

Use this file when the user needs the full transformation rollout path.

## Full sequence

1. Define the target state and rollout boundary.
2. Choose pilot scope and wave sequence.
3. Set governance, PMO cadence, and reporting.
4. Define enablement, audit, and reinforcement loops.
5. Track adoption and business outcome metrics.

## Review prompts

- What is safe enough to pilot first?
- Which dependency blocks the next rollout wave?
- Who resolves cross-functional transformation issues?
- How will adoption be measured beyond attendance and training completion?
