# Huawei Strategy Principles

## Overview

Apply an abstract strategy quality layer before detailed planning. Use this skill to improve thinking quality, not to replace the execution skills that produce the actual deliverables.

## Apply Principle Checks First

Judge strategic work against these questions:

- Is the plan anchored in business ambition and competitive reality, not just internal activity?
- Are the few decisive choices explicit?
- Are the capability gaps and control points visible?
- Is the annual plan derived from strategy instead of being a disconnected task list?
- Are metrics, milestones, and owners clear enough to operate?

Read [principle-checklist.md](references/principle-checklist.md) when a rigorous review is needed.

## Use The Skill In Three Modes

Use one of these modes depending on user intent:

- `Critique`: evaluate an existing strategy draft and flag weak logic
- `Shape`: help the user sharpen strategic choices before drafting
- `Guardrail`: stay present in the background while another execution skill is doing the heavy lifting

## Keep The Abstraction Useful

Do not drown the user in framework jargon. Explain weaknesses in plain business language:

- What choice is missing
- What evidence is weak
- What capability gap is ignored
- What annual action has no strategic basis
- What metric is absent or misaligned

## Hand Off Cleanly

If the user needs actual output instead of critique:

- Switch to `strategy-blm-dste-execution` for a strategy or annual plan draft
- Switch to `product-planning-execution` if the strategy is ready and roadmap work should start
