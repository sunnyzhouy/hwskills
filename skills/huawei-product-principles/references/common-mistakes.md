# Common Mistakes

Use this file when a roadmap looks busy but is not strategically or operationally strong.

## Common mistakes

- Treat every requirement as equally important
- Write a release plan with no theme or customer logic
- Ignore engineering and dependency constraints
- Hand off ambiguous scope to development

## Correction pattern

1. Re-anchor in strategy and customer scenario.
2. Group scope into a few release themes.
3. Force explicit tradeoffs.
4. Make assumptions, dependencies, and success criteria visible.
