# Common Misreads

Use this file when the team thinks leadership was clear but still risks acting on the wrong interpretation.

## Common misreads

- Treat a vision slogan as an immediate quarterly action
- Mistake emotional criticism for the whole problem definition
- Turn a local product comment into the total strategy
- Interpret "move faster" as pure effort instead of reprioritization
- Interpret "grow" as "go broad" while ignoring product or org constraints

## Correction pattern

1. Separate explicit instructions, repeated emphasis, and boundary-setting.
2. Test which of those signals would actually change resource allocation or roadmap choices.
3. Rewrite the conclusion as strategy, market, product, and org actions.
4. Mark low-evidence parts as open questions instead of pretending certainty.
