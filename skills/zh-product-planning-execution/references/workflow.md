# Workflow

Use this file when the user needs a full product planning path.

## Full sequence

1. Clarify the planning horizon and product boundary.
2. Identify target customer segments and scenarios.
3. Generate roadmap or release options.
4. Compare options by value, dependency, and feasibility.
5. Commit to release themes and top capabilities.
6. Prepare the handoff package for development.

## Planning prompts

- Which customer scenario creates the clearest business value?
- Which capabilities are truly differentiating?
- What can wait for a later release?
- Which dependency could break the release rhythm?
