# Workflow

Use this file when the user needs a complete IPD-oriented execution flow.

## Full sequence

1. Confirm approved scope, exclusions, and timing.
2. Decompose work into capabilities, workstreams, and milestones.
3. Define stage objectives and gate exits.
4. Identify cross-functional dependencies.
5. Build risk and quality controls.
6. Prepare steering and review materials.

## Review prompts

- What has to be true before the next gate?
- Which cross-team dependency is most fragile?
- What quality evidence is required before commitment?
- Which unresolved issue could derail the release?
