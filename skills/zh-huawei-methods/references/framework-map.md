# 方法关系图

当用户想理解整套方法之间的关系，而不是只解决一个具体交付物时，读取本文件。

## 分层

- `战略`：用 BLM、DSTE 一类方法定义方向、重点选择、控制点和年度优先级
- `市场`：做市场洞察、细分、目标市场选择和机会排序
- `产品规划`：把业务选择转成路标、版本和规划包
- `IPD 执行`：把已批准范围转成阶段化、跨部门的研发执行
- `销售`：把线索、商机、合同、交付、验收到回款连成闭环
- `服务`：把问题受理、定位、升级、解决、关闭和反馈连成闭环
- `变革`：把整套方法通过 PMO、治理和节奏推进到组织中

## 常见路径

1. 用户在问方向和资源聚焦时，从战略开始。
2. 用户要把战略落到路标和版本时，进入产品规划。
3. 用户要把批准范围落到多团队研发执行时，进入 IPD 执行。
4. 用户在重设计前端业务转化、商机治理、回款控制时，进入销售。
5. 用户在重设计售后问题闭环、升级、反馈时，进入服务。
6. 用户在问“整套体系怎么推开”时，进入变革。

## 快速诊断

- 用户问“该押什么方向”，从战略开始。
- 用户问“哪些能力放进哪个版本”，从产品规划开始。
- 用户问“跨团队怎么把这件事做成”，从 IPD 执行开始。
- 用户问“怎么把线索做到回款”，从销售开始。
- 用户问“怎么把客户问题真正闭环”，从服务开始。
- 用户问“怎么把这套方法落地推广”，从变革开始。
