# Workflow

Use this file when the user needs the full issue-to-resolution flow.

## Full sequence

1. Define issue classes and service scope.
2. Design intake and triage.
3. Set ownership and escalation rules.
4. Define real closure criteria.
5. Build feedback loops to product, quality, and management.
6. Set KPIs and review cadence.

## Review prompts

- How is issue severity classified?
- When does escalation become mandatory?
- What evidence proves the issue is resolved?
- Which problems must feed back into product or quality?
